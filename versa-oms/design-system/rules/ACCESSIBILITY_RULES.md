# Accessibility Rules

## Required

- Semantic HTML.
- One h1 per page.
- Keyboard navigation.
- Visible focus states.
- ARIA labels for icon-only buttons.
- Form labels associated with inputs.
- Error text tied to fields.
- Modal focus trap.
- Escape closes modal/drawer where safe.
- Color plus text/icon for status.
- Contrast-aware color choices.

## Operational Accessibility

- Status badges must include text.
- Risk labels must include text.
- Audit timeline must be screen-reader readable.
- Tables must use proper headers.
- Export/download buttons must describe file type and risk.

## Forbidden

- Color-only status.
- Icon-only high-risk actions.
- Auto-dismiss critical errors.
- Focus loss after validation failure.
