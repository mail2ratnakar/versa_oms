# Public Verification UI Rules

## Purpose

Public verification pages verify certificate/result authenticity with minimal exposure.

## Allowed Display

- verification status.
- certificate number.
- exam name.
- school name if policy allows.
- issue date.
- award/category if policy allows.
- revoked/reissued status.

## Forbidden Display

- parent contact.
- student contact.
- payment status.
- internal notes.
- raw score unless policy explicitly enables.
- OMR data.
- answer key.
- private file URL.
- audit information.

## UI Rules

- Simple search/lookup.
- Clear status.
- No login required.
- Rate limit.
- No broad search.
- Exact verification code only.
