# School Portal UI Rules

## Purpose

The school portal should be simple, clear and mobile-friendly.

## Design Rules

- Use plain language.
- Reduce internal terms.
- Use cards over dense tables on mobile.
- Show next required action clearly.
- Use progress/status panels.
- Show only own-school data.
- Avoid staff/internal audit terminology.
- Offer support ticket path.

## School Portal Main States

- onboarding pending.
- onboarding approved.
- roster pending.
- roster submitted.
- roster locked.
- payment pending.
- slot confirmed.
- materials available.
- results published.
- certificates available.
- blocked/suspended.

## School UI Must Not

- Show other schools.
- Show internal notes.
- Show payment provider payload.
- Show raw audit events.
- Show hidden material URLs.
- Show result correction internals.
