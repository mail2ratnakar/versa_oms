# 11 — Brain + Skills + Packs Operating Model

## Layer Model

```text
BRAIN
↓
SKILLS
↓
PACKS
↓
CODE
↓
TESTS
↓
SUMMARY
```

## Brain

Teaches how the founder thinks.

Use brain to understand:

- what complete means.
- how to detect gaps.
- how to fix bugs.
- when to stop.
- how to self-review.

## Skills

Teach how to work.

Use skills to apply:

- effect chain.
- screen contract.
- journey test.
- API contract.
- database migration.
- RBAC.
- security/privacy.
- audit/idempotency.
- signed URLs.
- workers.
- observability.
- CI/deployment.
- verification.

## Packs

Define what to build.

Use packs as source of truth.

## Code

Implement one pending item at a time.

## Tests

Prove the journey.

## Summary

Update completed/pending after every meaningful slice.

## Start Rule

Before any coding session:

1. read `spec/brain`.
2. read `spec/skills`.
3. read latest completed/pending summary (reports/BUILD_STATUS.md).
4. read relevant pack/module specs.
5. identify next pending item.
6. implement only that slice.
